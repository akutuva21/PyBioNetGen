## 2025-04-09 - Avoid List Comprehensions for Early Return Membership Checks
**Learning:** List comprehensions require evaluating all elements of an iterable, allocating a new list, populating it, and then iterating over it. This has `O(N)` time complexity and `O(N)` memory overhead, defeating short-circuiting. Replacing `return item in [x.name for x in items]` with an explicit `for` loop avoids these allocations and returns `True` immediately on match. While `any()` with a generator expression (`any(x.name == item for x in items)`) is more concise, an explicit `for` loop avoids generator initialization overhead and is slightly faster in performance-critical Python code.
**Action:** When performing membership checks that require accessing an attribute of elements in a list, always use an explicit `for` loop (or `any()` if conciseness outweighs the micro-optimization) to allow for early termination and avoid unnecessary list allocations.
