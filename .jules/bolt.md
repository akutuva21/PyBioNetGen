## 2024-05-18 - Optimize list comprehension to set membership in loop
**Learning:** Checking for membership in a list comprehension inside a loop (`[x for x in list]`) results in O(N^2) complexity because the list is recreated every iteration. Caching the elements in a `set` before the loop reduces complexity to O(N). If the loop logic can modify the list (like `append`), the cached `set` must be explicitly updated to maintain correct functionality and avoid logical regressions (e.g., adding duplicates).
**Action:** When optimizing loop logic involving membership tests against a collection that might be modified during the loop, cache the collection as a `set` *before* the loop and ensure the `set` is updated concurrently with the original collection to preserve correctness.
